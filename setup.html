Prerequisites & Setup

Create your accounts and install the tools you need before building.

Create Your Accounts

You need three free accounts. Take a few minutes to sign up and verify your email for each.

GitHub Account

What it is: A site where developers store and share code.

Why you need it: Your Harvest Hub code will live here, and GitHub connects to Vercel so your app can deploy automatically when you push changes.

Steps: Go to github.com → Sign up → Verify your email.

Choose a professional username—it will be part of your app’s URL when you use GitHub Pages or share your repo.

Vercel Account

What it is: A service that hosts web apps (puts them on the internet).

Why you need it: Vercel will take your code from GitHub and make it available to anyone with the link.

Steps: Go to vercel.com → Sign up with GitHub → Authorize access.

The free “Hobby” plan is all you need.

InstantDB Account

What it is: A backend service that stores data and handles user authentication.

Why you need it: Your app needs somewhere to store reservations, user accounts, and order data.

Steps: Go to instantdb.com → Sign up → Verify your email.

Create Your InstantDB App

This step is critical. Your app will connect to InstantDB using a unique identifier called an App ID.

  1. Go to https://instantdb.com/dash (the InstantDB dashboard).
  2. Click “Create App”.
  3. Name your app (e.g. Harvest Hub).
  4. You’ll see your App ID—a long string of letters, numbers, and dashes, like:
566ffee5-e773-4b68-a132-29678f8f6eea

What is the App ID?

Think of the App ID like a mailing address for your database. When your app reads or writes data, it needs to know where to send the request. The App ID tells InstantDB: “this request is for this app’s data, not someone else’s.”

The App ID is public—it’s safe to include in your code. It’s not a secret like a password. InstantDB uses a separate permissions system to control who can read or write what.

  1. Copy your App ID and save it somewhere (a notes app or text file). You’ll paste it into the build prompt in the next page.
  2. Keep the InstantDB dashboard tab open—you’ll return to it when you configure permissions.

Install Node.js

What is Node.js?

Website code usually runs in the user’s browser. During development, you also need to run code on your own computer—to preview the app, install packages, and prepare it for deployment. Node.js lets your computer understand and run JavaScript (the language web apps are built with).

Think of Node.js as a translator that lets your machine run the same language the browser uses.

  1. Go to https://nodejs.org.
  2. Download the LTS version (Long Term Support—the stable, recommended one).
  3. Run the installer and follow the prompts.
  4. To verify: open your terminal and type node --version. You should see a version number.

What is a terminal?

The terminal (or “command line”) is a text-based way to give instructions to your computer. Instead of clicking icons, you type commands. It can look intimidating, but you’ll only need a few commands, and Cursor will often run them for you.

On Mac: Open the “Terminal” app (search in Spotlight). On Windows: Open “PowerShell” or “Command Prompt” (search in the Start menu).

Install Cursor

What is Cursor?

Cursor is a code editor—a text editor built for writing code. Unlike a normal editor, it has a built-in AI assistant. You describe what you want in plain English, and Cursor can write and change the code for you. You’ll be using Cursor to direct the AI, not to type code yourself.

  1. Go to https://cursor.com.
  2. Download and install Cursor.
  3. Open Cursor and complete the setup wizard.
  4. Sign in or create an account when prompted.

Configure Cursor for This Project

Why This Step Matters

Cursor’s AI is trained on lots of code and docs, but it has a “knowledge cutoff”—it may not know the latest details for newer tools like InstantDB. By adding InstantDB’s documentation to Cursor, you give the AI a reference it can use while helping you, which improves the code it generates.

When to add docs: For newer or less common tools (like InstantDB), or when you get wrong or outdated suggestions.

When it’s not necessary: For very popular tools (e.g. React, basic JavaScript) that the AI already knows well.

  1. Open Cursor Settings (Ctrl+, on Windows, Cmd+, on Mac).
  2. Go to Features → Docs.
  3. Click “Add Doc”.
  4. Enter this URL: https://www.instantdb.com/docs
  5. Wait for Cursor to finish indexing (may take a minute).
  6. You should see InstantDB listed in your docs.
