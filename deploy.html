Deployment

Get your Harvest Hub app from your computer onto the internet.

What is Deployment?

Right now, your app runs only on your computer. If you close your laptop, no one can use it. Deployment means putting your app on a server—a computer that’s always on—so anyone with the URL can use it anytime.

We’ll use Vercel because:

  • It’s free for personal projects.
  • It automatically redeploys when you update your code (push to GitHub).
  • It was created by the team behind Next.js, so they work together seamlessly.

Push Your Code to GitHub

Vercel will pull your code from GitHub, so first we put the project on GitHub.

  1. Go to github.com and sign in.
  2. Click the + in the top right → New repository.
  3. Name your repository (e.g. harvest-hub—lowercase, dashes for spaces).
  4. Keep it Public (required for free Vercel hosting).
  5. Do not check “Add a README” or “Add .gitignore”—your project already has these from Cursor.
  6. Click Create repository.
  7. You’ll see a page with instructions—keep this tab open.

Then, in Cursor:

  1. Open the terminal (View → Terminal).
  2. Ask Cursor: “Please help me push this project to GitHub. My repository URL is [paste the URL from GitHub].”
  3. Cursor will guide you through the git commands (e.g. git init, git add ., git commit, git remote add origin ..., git push).
  4. Refresh your GitHub page—you should see your files.

Deploy to Vercel

  1. Go to vercel.com and sign in (with GitHub).
  2. Click “Add New Project” (or “Import Project”).
  3. You’ll see your GitHub repos—select your Harvest Hub repo.
  4. Before deploying, add your InstantDB App ID:
    • Click “Environment Variables” (or expand that section).
    • Add: Name = NEXT_PUBLIC_INSTANT_APP_ID, Value = your InstantDB App ID.
    • Click Add.
  5. Click “Deploy”.
  6. Wait 1–2 minutes for the build to finish.
  7. When you see “Congratulations!”—click the URL to open your live app.

What are Environment Variables?

Some values (like your App ID) shouldn’t be hardcoded in files that are public on GitHub. Environment variables let you pass configuration into your app at build or runtime without putting secrets in the code.

The NEXT_PUBLIC_ prefix tells Next.js this variable is safe to expose to the browser. The InstantDB App ID is public by design; the prefix just makes it available to client-side code.

Your App is Live

Your Harvest Hub app is now on the internet. You can:

  • Share the URL with friends or classmates.
  • Open it on your phone or any device.
  • Use it even when your own computer is off.

Automatic updates: Whenever you push new code to GitHub, Vercel will detect the change, rebuild your app, and redeploy it. This usually takes 1–2 minutes. You don’t have to do anything else.
