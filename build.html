Building Your App

Use Cursor to generate your Harvest Hub app from one prompt.

Create Your Project Folder

  1. Create a new folder on your computer for this project. Name it something simple, like harvest-hub (no spaces—use dashes). Put it somewhere easy to find, like your Desktop or Documents.
  2. Open Cursor.
  3. Go to File → Open Folder and select your new folder.
  4. You should see an empty folder in Cursor’s sidebar.

The Build Prompt

Next, you’ll ask Cursor to build your entire app by pasting one long prompt.

Before you proceed

Make sure you completed the Setup page and have your InstantDB App ID copied. If not, go to your InstantDB dashboard, open your app, and copy the App ID.

  1. Open Cursor’s Composer (Ctrl+I on Windows, Cmd+I on Mac)—the chat where you describe what you want.
  2. Copy the entire prompt below (click the Copy button on the code block).
  3. Replace YOUR_APP_ID_HERE in the prompt with your actual InstantDB App ID (every place it appears).
  4. Paste the prompt into Composer and press Enter. Let Cursor work—it may take a few minutes.

Build prompt (copy the entire block):

Build me a Next.js web application. Here are the complete specifications:

## Tech Stack (DO NOT DEVIATE)
- Next.js 14+ with App Router (not Pages Router)
- TypeScript
- Tailwind CSS for styling
- InstantDB for database and authentication (use @instantdb/react)

## InstantDB Configuration
My InstantDB App ID is: YOUR_APP_ID_HERE

Initialize InstantDB in a file at `src/lib/db.ts` like this:

import { init } from '@instantdb/react';
import schema from '../../instant.schema';

const APP_ID = 'YOUR_APP_ID_HERE';

export const db = init({
  appId: APP_ID,
  schema
});

## App Description
Harvest Hub is a customer-facing web app where people can reserve a weekly produce bundle, choose a pricing tier (Equity, Anchor, or Steward) that supports the community capacity pool, read about the mission and sourcing, and track their order through a simple allocation process. This is reservation-only—no real payment processing. Users have accounts and see their own reservations and order status.

## Data Model
Create an InstantDB schema in `instant.schema.ts` with these entities:

- **orders**: timestamp (date), tier (string: "equity" | "anchor" | "steward"), status (string: e.g. "received" | "in_queue" | "assigned" | "confirmed" | "ready"), assignedNodeId (optional string, or link to nodes)
- **nodes**: name (string), capacity (number), remainingCapacity (number)

And these relationships:
- Each user ($users) has many orders
- Each order belongs to one user
- Each order can be assigned to one node (optional)

## Authentication
Implement user authentication using InstantDB's Magic Code flow:
1. User enters their email
2. InstantDB sends a 6-digit code to that email
3. User enters the code to log in
4. Use db.useAuth() to check login status
5. Protect routes that require authentication (reserve, my orders)

## Pages to Create
- `/` — Home page: hero, links to Reserve, Dashboard, Learn, and Track (or My Orders). Show login prompt if not authenticated.
- `/login` — Login page with email input and Magic Code verification
- `/reserve` — Weekly produce bundle description, tier selector (Equity / Anchor / Steward), Reserve button (writes order to InstantDB). Require auth.
- `/dashboard` — Community capacity pool dashboard: bundles funded this week, households supported, progress bar toward weekly capacity (can be derived from orders or mock data)
- `/learn` — Learn section with three subsections: "Why This Exists" (food access, transportation barriers, coordinated distribution), "Where Produce Comes From" (sourcing, local pickup nodes), "What is Devoir" (coordination system for fair distribution). Static content is fine.
- `/orders` or `/my-orders` — User's orders with status: Order Received → Entered Allocation Queue → Assigned to Pickup Node → Capacity Confirmed → Ready for Pickup. Optional transparency toggle showing tier, timestamp, pickup node, capacity snapshot.

## Core Features
1. Users can view the weekly produce bundle and reserve it (choose tier; no payment)
2. Community capacity pool dashboard showing bundles funded, households supported, progress
3. Learn section with the three subsections above
4. Order tracking with allocation-style statuses; optional transparency details
5. Mock allocation: when an order is created, simulate assignment (e.g. update status and assign to a node in InstantDB, or use a simple Next.js API route / server action that updates order status and node remainingCapacity)

## UI Requirements
- Clean, modern design using Tailwind CSS
- Mobile-responsive (works on phones and desktops)
- Clear navigation between all pages
- Loading states while data is being fetched
- Error messages when something goes wrong

## File Structure
Use this structure:

src/
├── app/
│   ├── layout.tsx        # Root layout with InstantDB provider
│   ├── page.tsx           # Home page
│   ├── login/
│   │   └── page.tsx       # Login page
│   ├── reserve/
│   │   └── page.tsx       # Reserve bundle
│   ├── dashboard/
│   │   └── page.tsx       # Capacity pool dashboard
│   ├── learn/
│   │   └── page.tsx       # Learn section (or sub-routes for each part)
│   └── orders/
│       └── page.tsx       # My orders / tracking
├── components/
│   └── [reusable components as needed]
└── lib/
    └── db.ts              # InstantDB initialization
instant.schema.ts          # InstantDB schema
instant.perms.ts           # InstantDB permissions

## Getting Started Steps
1. Create the Next.js app: npx create-next-app@latest . --tailwind --eslint --app --src-dir --import-alias "@/*" --typescript
2. Install InstantDB: npm install @instantdb/react
3. Create the schema, permissions, and all files listed above
4. Set up permissions so users can read/write their own orders and read nodes (and any global data you need)

Please create all necessary files with complete, working code. Replace YOUR_APP_ID_HERE with my actual InstantDB App ID everywhere in the prompt and in the generated code.

What’s in the Prompt?

The prompt tells Cursor exactly what stack to use (so it doesn’t pick something else), how to connect to your InstantDB app, what data to store, how users log in, which pages to build, and what features to include. Everything is tailored to Harvest Hub so the generated app matches your idea.

What to Expect

When you run the prompt:

  • Cursor will start creating files—you’ll see them appear in the sidebar.
  • It may take a few minutes; let it finish.
  • Cursor might ask clarifying questions—answer them based on your Harvest Hub idea.
  • When it’s done, you’ll have a full project structure.

Don’t worry if:

  • There are errors at first—you’ll fix them with Cursor’s help.
  • The code looks overwhelming—you don’t need to understand every line.
  • Cursor asks you to run terminal commands—follow its instructions.

Running Your App Locally

Once the project is generated, run it on your computer:

  1. Open the terminal in Cursor (View → Terminal, or Ctrl+`).
  2. Type this command and press Enter:
npm run dev
  1. Wait for it to compile (you’ll see “Ready” and a URL).
  2. Open your browser and go to http://localhost:3000.
  3. You should see your app.

What is localhost?

When you run your app in development, it’s not on the internet yet—it runs only on your computer. localhost is a special address meaning “this computer.” :3000 is the port—like a specific door your app is using.

Fixing Issues with Cursor

If something doesn’t work:

  1. Look at the error message (in the browser or in the terminal).
  2. Copy the error message.
  3. Open Cursor’s Composer (Ctrl+I / Cmd+I).
  4. Paste the error and ask: “I’m getting this error. Please fix it.”
  5. Cursor will suggest changes—apply them and try again.

You don’t need to understand the error yourself; you just need to show it to the AI.
